What Is a Large Language Model?

An LLM is a neural network trained on massive amounts of text to predict the next token — a word piece, punctuation mark, or code character.

Three things to remember:

ConceptWhat It Means
TrainingHappened once, on internet-scale text. The model's "knowledge" is frozen at a cutoff date.
InferenceHappens every time you use it. The model generates tokens one by one, in real time.
Context windowThe working memory — everything the model can "see" in one conversation. Measured in tokens.

Hallucinations happen when the model generates plausible-sounding tokens that aren't grounded in fact. This is a core behavior, not a bug to be patched.

Claude — Anthropic

Model family (2025): Claude Opus 4 · Claude Sonnet 4 · Claude Haiku 4

Claude is Anthropic's frontier model, designed with a strong focus on safety, long-context reasoning, and agentic tool use.

Surfaces:

  • Claude.ai — web chat, Projects (persistent file context), image/PDF understanding
  • Claude Code — terminal-based coding agent (like a junior developer in your shell)
  • Claude Desktop — desktop app with MCP server support
  • Anthropic API — integrate Claude into any application

What you can build:

  • Skills / slash commands (.claude/commands/*.md) — reusable prompt templates
  • CLAUDE.md — project-level behavioral rules and context
  • MCP Servers — TypeScript or Python servers that give Claude new tools, data, and prompts
  • Claude Code SDK — script programmatic agents that read files, run commands, call APIs
  • Marketplace plugins — bundles of skills + MCP server + config for team distribution
  • Prompt-cached pipelines for cost-efficient repeated workloads
This workshop focuses here. Claude + MCP is the most extensible stack for technical writers.

ChatGPT — OpenAI

Model family (2025): GPT-4o · GPT-4o mini · o1 · o3 · o4-mini

OpenAI pioneered the modern LLM era. GPT-4o is their flagship multimodal model; the o-series models (o1, o3, o4-mini) use extended chain-of-thought reasoning for harder problems.

Surfaces:

  • ChatGPT.com — web and mobile chat interface
  • OpenAI API — programmatic access for developers
  • OpenAI Assistants API — stateful agents with threads
  • OpenAI Codex CLI — open-source terminal coding agent (like Claude Code)

What you can build:

  • Custom GPTs — no-code AI assistants with custom instructions, uploaded files, and Actions
  • GPT Actions — connect a Custom GPT to any REST API via an OpenAPI spec
  • Assistants — agents with persistent threads, file search, code interpreter, function calling
  • Fine-tuned models — train GPT-4o mini on your proprietary data and style
  • Structured outputs — guarantee JSON schema-compliant responses for automation
  • Batch API pipelines — cost-efficient bulk document processing
Best for: teams already in the OpenAI ecosystem, Custom GPT distribution on GPT Store.
7 / 17

Google Gemini

Model family (2025): Gemini 2.5 Pro · Gemini 2.0 Flash · Gemini 1.5 Pro (2M token context)

Google's multimodal frontier model — understands text, images, video, audio, and code natively.

Surfaces:

  • Gemini.google.com — web interface, Gems (custom AI assistants)
  • Google AI Studio — rapid prototyping and API exploration
  • Gemini in Google Workspace — AI inside Docs, Sheets, Slides, Gmail, Meet
  • Vertex AI — enterprise deployment with governance, grounding, and RAG
  • NotebookLM — AI research assistant grounded in your uploaded documents

What you can build:

  • Gems — custom-configured Gemini assistants for specific personas or tasks
  • Grounded apps — connect to Google Search or your own data sources for real-time accuracy
  • Vertex AI agents — production agents with memory, tools, and enterprise security
  • Workspace add-ons — extend Docs/Sheets/Slides with Gemini API calls
  • Multimodal pipelines — process PDFs, images, video frames, and audio in one API call
  • Function calling — same tool-use pattern as OpenAI, with long-context advantages
Best for: Google Workspace shops, long-document analysis (2M context), multimodal use cases.

GitHub Copilot — Microsoft / GitHub

Copilot is not a single model — it routes to OpenAI, Anthropic, and Google models depending on the task. It's a developer productivity layer built on top of frontier models.

Surfaces:

  • IDE extensions — VS Code, JetBrains, Neovim, Visual Studio (inline suggestions + chat)
  • Copilot Chat — conversational AI inside the IDE, GitHub.com, and Mobile
  • Copilot CLI — AI-assisted terminal commands
  • Copilot Workspace — task-based agentic development environment (web)
  • GitHub.com — PR summaries, code review explanations, issue descriptions

What you can build:

  • Copilot Extensions — GitHub Apps that integrate with Copilot Chat:

- Skillsets: REST endpoints that answer domain-specific questions

- Agents: full conversational agents with action-taking capabilities

  • Custom instructions — .github/copilot-instructions.md to tune behavior per repo
  • Copilot Workspace sessions — guided multi-step agentic coding flows
Best for: development teams where the IDE is the primary work surface.
9 / 17

Microsoft 365 Copilot

Microsoft 365 Copilot embeds AI into the entire Office productivity suite. It runs on GPT-4 class models via Azure OpenAI.

Surfaces:

  • M365 Apps — Word, Excel, PowerPoint, Outlook, Teams, OneNote
  • Copilot.microsoft.com — standalone chat (consumer + enterprise)
  • Copilot Studio — low-code platform to build custom Copilot agents
  • Power Platform — Power Automate + Power Apps with Copilot integration

What you can build:

  • Declarative agents — JSON-manifest extensions to M365 Copilot with custom instructions, SharePoint grounding, and API connectors
  • Custom copilots (Copilot Studio) — no-code/low-code bots for Teams, web, or any channel
  • API plugins — connect M365 Copilot to external systems via OpenAPI spec (same pattern as GPT Actions)
  • Teams message extensions — Copilot-queryable cards surfaced in Teams and Outlook
  • Power Automate flows — triggered by Copilot chat to automate processes
Best for: enterprises deeply invested in Microsoft 365 who want AI inside existing workflows.
10 / 17

Cursor — The AI-First IDE

Cursor is a VS Code fork rebuilt around AI. All standard VS Code extensions work. The AI layer is deeply integrated at every level.

Key AI features:

  • Tab — inline next-line and multi-line completions (faster than Copilot, context-aware)
  • Chat — conversational AI with @file, @docs, @web, @codebase context injection
  • Composer / Agent — multi-file edits; in Agent mode, Cursor reads errors, runs commands, iterates
  • MCP support — connect any MCP server to Cursor for tool and data access

What you can configure / build:

  • Cursor rules — .cursor/rules/*.mdc files (like Claude's CLAUDE.md) to set AI behavior per project
  • MCP server connections — give Cursor access to your files, APIs, and systems
  • Custom doc indexing — add any URL or file set as @docs context
  • Model selection — choose Claude Sonnet, GPT-4o, Gemini Flash per session

Limitation: Cursor's extensibility is config-level. You define rules and connect MCP servers; you don't build new surfaces inside Cursor the way you build skills for Claude Code.

Best for: developers who want a full IDE with deep AI integration out of the box.

Claude Desktop — Your MCP-Enabled AI Workstation

Claude Desktop is Anthropic's official desktop app (macOS + Windows). It looks like Claude.ai but has one critical superpower: native MCP server support.

What makes it different from Claude.ai:

  • Connect MCP servers by editing claude_desktop_config.json
  • Claude can then read your local files, query databases, call APIs — without you copy-pasting
  • Supports Computer Use (experimental): Claude can control your screen
  • Projects: upload files that persist as shared context across all conversations

Configuration pattern:

{
  "mcpServers": {
    "doc-quality-checker": {
      "command": "node",
      "args": ["/path/to/your/server/dist/index.js"]
    }
  }
}

What you can build for Claude Desktop:

  • Any MCP server that exposes tools, resources, or prompts
  • Once connected, Claude Desktop users get those capabilities without any code
Best for: non-developer team members who need MCP-powered Claude without a terminal.
12 / 17

OpenAI Codex CLI — Terminal Coding Agent

Codex CLI is OpenAI's open-source terminal coding agent, launched in 2025 as a direct competitor to Claude Code.

How it works:

  • Runs in your terminal — reads your project, writes files, runs commands
  • Powered by GPT-4o and o4-mini
  • Three approval modes:
ModeBehavior
Suggest (default)AI proposes changes; you approve each step
Auto-editReads and writes files automatically; asks before running commands
Full-autoFully autonomous — runs commands and edits without approval

Key differences from Claude Code:

  • Open source (Apache 2.0 license) — you can fork and modify it
  • Uses OpenAI models by default (configurable)
  • No equivalent to Claude's skill system or marketplace integration
  • Sandboxed execution on macOS and Linux (Docker on Linux)
Use Codex CLI if your team is already on OpenAI and wants a terminal agent to pair with Custom GPTs.

Groq — Ultra-Fast AI Inference

Groq is not an AI model. It is an inference platform built on custom silicon called the LPU (Language Processing Unit) — hardware designed specifically to run LLMs at extreme speed.

What Groq offers:

  • GroqCloud API — run open-source models at 300–800 tokens/second (vs ~30–100 on standard GPU)
  • Models available: Llama 3.3 70B, Llama 3.1 405B, Mixtral 8x7B, Gemma 2, Whisper (transcription)
  • OpenAI-compatible endpoints — swap openai for groq in your code, same API shape

When to use Groq:

  • Real-time transcription or voice pipelines (Whisper at high speed)
  • Latency-sensitive documentation workflows where response time matters
  • Cost-efficient bulk runs on open models
  • Prototyping with open-source models before committing to a commercial API

What you can build:

  • Any application you'd build on OpenAI, using the same API pattern
  • Low-latency documentation review pipelines
  • Real-time style checking during authoring
Think of Groq as a turbo engine for open-source models — not a model in itself.

Choosing the Right Tool

"I want to automate a documentation review task"
  → Claude Code skill (Reviewer pattern)

"I want non-technical teammates to run the review tool"
  → Claude Desktop + MCP server

"I want to integrate AI into our Microsoft Teams workflow"
  → M365 Copilot declarative agent or Copilot Studio

"I want AI inline while I write code"
  → Cursor or GitHub Copilot in VS Code

"I want to build a public AI tool and share it on a marketplace"
  → Custom GPT (GPT Store) or Claude Marketplace plugin

"I want the fastest possible inference for a real-time pipeline"
  → Groq with an open-source model

"I want AI to process documents across Google Drive and Workspace"
  → Gemini + Vertex AI with Workspace integration

Why We're Building on Claude + MCP

Three reasons this stack was chosen for technical writers:

1. Skills are markdown files.

You already write markdown. A skill is a well-structured instruction set — and writing clear instructions is your job. No coding required to build useful, reusable AI tools.

2. MCP is the open standard.

Model Context Protocol is backed by Anthropic but implemented across Claude, Cursor, Zed, and other tools. What you build here is portable — your MCP server works in Cursor too.

3. Technical writers are the right people to build these tools.

Prompt quality drives output quality. A writer who understands audience, structure, and constraints will build better skills than a developer who doesn't think about those things. This stack puts that advantage to work.

By the end of this program...
You write skills that review, generate, transform, and analyze docs
You build an MCP server that your whole team can use
You understand how to extend the same pattern to ChatGPT, Cursor, or M365
